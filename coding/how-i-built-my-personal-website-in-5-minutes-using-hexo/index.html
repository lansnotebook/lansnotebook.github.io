Why Hexo?

Hexo is a static site generator, which is a fancy way of saying:

You write words → Hexo turns them into a website

Why I love it:

  • Lightweight
  • Markdown-based (aka writing without UI drama)
  • Perfect for blogs, journals, portfolios, and "I just want to exist online" websites
  • Customizable later (important keyword: later)

If your goal is sharing thoughts, not fighting CSS, Hexo is a great friend.

为什么选择Hexo？

Hexo 是一个静态网站生成器，简单来说就是：

你写点东西 → Hexo 把它们变成一个网站

我喜欢它的原因：

  • 轻量级

  • 基于 Markdown（也就是写作时无需忍受界面的繁琐）

  • 非常适合博客、日志、作品集以及“我只想在网上存在”之类的网站

  • 可自定义（重点：可自定义）

如果你的目标是分享想法，而不是与 CSS 作斗争，那么 Hexo 就是你的好帮手。

你需要准备的东西（非常简短的清单）

在我们召唤网站之神之前，请确保你已安装：

  • Node.js

  • Git

  • 终端（别害怕，它只是一个充满自信的文本框）

就这些。没有 27 步的清单。

步骤 1：安装 Hexo（30 秒）

在终端中输入：


npm install -g hexo-cli

恭喜！您已成功安装 Hexo，正式成为一名“终端用户”。

步骤 2：创建您的网站（1 分钟）


hexo init my-blog

cd my-blog

npm install

此时，Hexo 已悄然创建了：

  • 博客结构

  • 名称神秘的文件夹

  • 存放您想法的未来

步骤 3：本地运行（30 秒）


hexo server

打开浏览器并访问：

👉 http://localhost:4000

搞定！

您现在拥有一个网站了。

已经！

是的，就是这么快！

第四步：写点什么（有趣的部分）

创建你的第一篇文章：


hexo new post "Hello World"

使用 Markdown 编辑 source/_posts 文件夹中的文件。

没有干扰。

没有格式焦虑。

只有文字。

非常适合：

  • 日记

  • 项目笔记

  • 深夜哲学思考

  • “我的人生目标是什么”之类的文章

第五步：部署（可选，但很神奇）

你可以部署到 GitHub Pages、Netlify 或任何支持静态网站的平台。

部署完成后，你的个人网站就正式存在于互联网上了——仔细想想，这感觉还挺奇妙的。
